## Locked delivery model (customer flow)

At Pylon team registration on `pylonfootball.com` (or on a Pylon-branded TOJ-hosted
checkout the redirect points to), each team sees an **opt-in packages step**:

```
YOUR PYLON 2026 REGISTRATION IS CONFIRMED.
Optional add-ons:

□ Pylon Team Recap ($299 / event)         → Season Pack (best value $1,499)
□ Pylon Team Analytics ($499 / season)    → Head coach film + game planning
□ Pylon Combine Book ($2,999 / season)    → Every athlete graded + printed book
□ Pylon Recruiting Package (line-item TBD) → athlete-level Subject Report profiles
```

**Team package vs event package** — per Kyron: teams can buy either a per-event
add-on or a season pack. Both unlock the ability to purchase **Pylon Recruiting
Packages** (per-athlete Subject Report + On-Field Transcript) OR **Media
Packages** (hype videos, social content) on top. Checkout order:

1. **Base tournament registration** (Pylon takes their existing per-team fee)
2. **Team package** (Media or Analytics or Combine Book) — added on
3. **Per-athlete Recruiting Package** — athletes on rostered teams can be
   upgraded individually to a Subject Report profile + season grading
4. **Per-athlete Hype/Social Media Package** — targeted content bundle for the
   athletes with recruiting momentum

**Video crew allocation:**
- Recap tier + Season Pack → **Subject Media mentorship crew** (event-level)
- Combine Book + Recruiting Package + high-value events → **Subject Media contract crew** (event-contracted or year-contracted)

#### 2b. Routes

**File:** `routes/pylon.js`

Endpoints:
- `GET /api/pylon/teams` — search + list (query `?q=&region=&season=`)
- `POST /api/pylon/teams` — upsert
- `GET /api/pylon/events` — list (query `?season=&type=`)
- `POST /api/pylon/events` — upsert
- `POST /api/pylon/registrations` — record a team-event registration
- `POST /api/pylon/packages/quote` — takes `{ team_id, sku, event_id?, athlete_ids? }` → returns line-item pricing + all four revenue splits
- `POST /api/pylon/packages/order` — creates the order row + returns Stripe checkout URL
- `POST /api/pylon/packages/webhook` — Stripe webhook to mark order paid + trigger delivery workflow
- `GET /api/pylon/dashboard` — Kyron's overview: MTD/YTD gross, TOJ take, top teams, sponsor pipeline
- `POST /api/pylon/sponsors/deals` — create sponsor deal
- `PATCH /api/pylon/sponsors/deals/:id` — status transitions (pending → signed → paid)

Auth: session cookie for the admin surface; `X-Pylon-API-Key` header
(`PYLON_API_KEY` env) for anything Pylon corp calls server-to-server. Mirror the
pattern already used in `routes/hooks-agents.js`.
