### Design language

- Black + yellow (Pylon's palette from the site drop): `--pylon-black:#0A0A0A`, `--pylon-yellow:#F4C82E`, `--pylon-white:#FFFFFF`
- Display face: sans-serif condensed (Anton or similar — match Pylon's actual site typeface)
- Body face: modern sans (Inter or system stack)
- No serif on the Pylon-branded pages (reserve Fraunces for TOJ's own marketing site)
- Card treatment: sharp corners, thin yellow accent rule on hover
- Match the visual language of `pylonfootball.com` — do NOT reuse TOJ's cream/navy from `foundation-score.html`

Reference the actual Pylon site typography + palette when styling these three pages. If Pylon's site uses specific brand fonts, load them as woff2 with @font-face (no Google Fonts fetch at runtime).

### Delivery pipeline (post-purchase flow)

1. Stripe checkout completes → `/api/pylon/packages/webhook` fires
2. Order row created in `pylon_package_orders` (V2 schema) with status `pending_delivery`
3. Creator assignment logic runs:
   - Look up assigned creators for `event_id` in `pylon_media_assignments` (new table below)
   - If assignment exists → set `package_order.assigned_creator_email`
   - If not → land in `unassigned_queue` for the Head of Automation/Ops to resolve
4. Confirmation email fires to parent/coach: "Your Pylon Media order is confirmed. Your creator: [Name]. Expected delivery: 72 hours post-event."
5. Post-event: creator delivers assets → uploads to `pylon_package_orders.delivery_urls[]` → status `delivered`
6. Delivery notification email fires with links

### Additional migration to append to `db/migrations/008_pylon_partnership.sql`

```sql
create table pylon_media_assignments (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references pylon_events(id) on delete cascade,
  team_id uuid references pylon_teams(id) on delete set null,
  athlete_id uuid references pylon_athletes(id) on delete set null,
  creator_email text not null,
  creator_name text not null,
  content_taxonomy_slot text not null,     -- 'content_1' | 'content_2' | 'content_3a' | 'content_3b' | 'content_3c' | 'team_highlight'
  status text default 'assigned',           -- 'assigned' | 'shot' | 'edited' | 'delivered' | 'reassigned'
  briefed_at timestamptz,
  shot_at timestamptz,
  delivered_at timestamptz,
  qc_score int,                             -- 1-5 against Visual Direction rubric from V2.8
  qc_notes text,
  metadata jsonb default '{}'::jsonb,
  created_at timestamptz not null default now()
);
create index pylon_media_assignments_event_idx on pylon_media_assignments(event_id, status);
create index pylon_media_assignments_creator_idx on pylon_media_assignments(creator_email, briefed_at desc);

alter table pylon_package_orders
  add column if not exists assigned_creator_email text,
  add column if not exists delivery_urls text[] default '{}';
```

## Vendor booth staffing — the tiered model

Codified as `skills/booth-staffing/SKILL.md` in pylon-os. Booth staffing scales
with event size.

| Event size | Team count | Booth staffing | Cost profile |
|---|---|---|---|
| **Small** | 32–60 teams | 1–2 creators rotate booth duty (60/30 split shooting/booth) | Zero incremental cost — creators are already onsite for content |
| **Mid** | 60–120 teams | 1 dedicated Pylon Media seller + 3–5 creators for shooting | Seller comp: $500/event OR 15% of booth-attributable sales |
| **Large** | 120–265 teams | 2 dedicated sellers + 6–8 creators + 1 booth manager | Sellers: $500/event each · Manager: $750/event · 20% of sales pooled for creator tips |

### The dedicated seller role (Mid + Large events)

New role, contracted per-event or per-season by the Content Pod's Head of
Automation/Ops. Runs the 6-step Player Package Sales Guide (V2.8) at the booth.

Not a creator — a seller. Their entire job at the event is closing Player
Package sales walk-up. QR code + sample loop on booth screen + tablet checkout.

**Recommended profile:** ex-D3 or ex-JUCO athlete with parent-facing rapport,
football fluency, and comfort with sales. Compensation weighted toward
performance (base + %).

### The booth manager role (Large events only)

Coordinates the seller team + the creator team + the vendor booth logistics
(setup, samples, connectivity, restock). Reports to the Head of Automation/Ops
during the event.

create table pylon_booth_staffing_plans (
  id uuid primary key default gen_random_uuid(),
  event_id uuid not null references pylon_events(id) on delete cascade,
  event_size_tier text not null,                       -- 'small' | 'mid' | 'large'
  dedicated_sellers_planned int default 0,
  creators_planned int default 0,
  booth_manager_planned boolean default false,
  actual_sellers_present int default 0,
  actual_creators_present int default 0,
  booth_manager_present boolean default false,
  gross_booth_sales_cents int default 0,
  planned_at timestamptz not null default now(),
  event_completed_at timestamptz
);
create index pylon_booth_staffing_plans_event_idx on pylon_booth_staffing_plans(event_id);
