## Locked decisions (V2.5)

1. **TOJ owns all four content pillars.** Ritual + Scale + Prestige + Capture. Single system, single ownership.
2. **Pylon Path anchors are drawn from represented athletes** (kids with existing marketing/agent infra — faster consent, cleaner IP path). Casting comes from Pylon's signed roster + represented-partner rosters.
3. **All content speaks with Pylon's voice.** TOJ builds the system; Pylon appoints the human overseer (the Creative Lead). TOJ's fingerprint stays invisible in published assets.
4. **Content Pod = staffed team.** Creative Lead + reports. Not a solo operator.

### 2l. Additional routes — append to `routes/pylon.js`

Content calendar + pieces:
- `GET /api/pylon/content` — filter by `?pillar=&status=&scheduled_before=&scheduled_after=`
- `POST /api/pylon/content` — create piece (draft)
- `PATCH /api/pylon/content/:id` — edit / advance status
- `POST /api/pylon/content/:id/publish` — mark published + record initial performance
- `GET /api/pylon/content/calendar?from=&to=&pillar=` — calendar-view feed
- `POST /api/pylon/content/:id/performance` — ingest engagement snapshot

Ambassadors:
- `GET /api/pylon/ambassadors` · `POST /api/pylon/ambassadors` (upsert) · `PATCH /api/pylon/ambassadors/:id`
- `POST /api/pylon/ambassadors/:id/posts` — log a post + trigger amplification workflow
- `GET /api/pylon/ambassadors/:id/compliance` — post-cadence compliance report

Newsletter:
- `POST /api/pylon/newsletter/subscribe` — free tier signup (open)
- `POST /api/pylon/newsletter/upgrade` — premium checkout via Stripe → returns Stripe URL
- `POST /api/pylon/newsletter/webhook` — Stripe recurring billing webhook
- `GET /api/pylon/newsletter/issues` · `POST /api/pylon/newsletter/issues` — issue editor
- `POST /api/pylon/newsletter/issues/:id/send` — queue delivery via existing `utils/email` infrastructure

Content performance rollup:
- `GET /api/pylon/content/performance/by-pillar?season=&week=` — cross-pillar rollup for dashboards
- `GET /api/pylon/content/performance/top?limit=10&pillar=` — top-performing pieces

Voice-authority guardrail (server-side check on every publish):
- `POST /api/pylon/content/:id/approve` — requires session cookie belonging to a Pylon-designated approver (checked against `PYLON_APPROVERS` env var array). Non-approver approvals return 403.
